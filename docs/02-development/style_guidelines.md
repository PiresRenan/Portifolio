# Style Guidelines

This document outlines the coding conventions and style guidelines to be followed in the Fullstack Developer Portfolio project. Consistent code style improves readability, maintainability, and collaboration.

## General

*   Use 2 spaces for indentation.
*   Limit line length to 80 characters.
*   Use meaningful names for variables, functions, and classes.
*   Write clear and concise comments to explain the code's purpose and functionality.

## Frontend (React/TypeScript)

*   Follow the official React and TypeScript style guides.
*   Use functional components and hooks whenever possible.
*   Prefer arrow functions over traditional function declarations.
*   Use template literals for string interpolation.
*   Use JSX for rendering UI elements.

## Backend (Java)

*   Follow the official Java style guide.
*   Use camelCase for variable and method names.
*   Use PascalCase for class names.
*   Use meaningful names for packages and modules.

## Backend (Python)

*   Follow the PEP 8 style guide.
*   Use snake_case for variable and function names.
*   Use PascalCase for class names.
*   Use meaningful names for modules and packages.

## Testing

*   Write unit tests for all components, functions, and classes.
*   Use a testing framework like Jest (for frontend) or Pytest/JUnit (for backend).
*   Strive for high test coverage.
